IOS基础-UIWindow和UIView

By yanlc

发表于 2015-09-09

UIWindow

1.UIWindow继承自UIView， 虽说UIWindow 继承于UiView， 但这个 UIWindow 也是一个重中之重的View。 这里，打个比方： UIWindow 虽说也是个View，但它是一个首席View。
UIWindow对象是所有UIView的根，管理和协调的应用程序的显示
UIWindow类是UIView的子类，可以看作是特殊的UIView。一般应用程序只有一个UIWindow对象，即使有多个UIWindow对象，也只有一个UIWindow可以接受到用户的触屏事件。
UIWindow的主要作用：一是提供一个区域来显示UIView，二是将事件（event）的分发给UIView

AppDelegate.m

-(BOOL)application:(UIApplication)application didFinishLaunchingWithOptions:(NSDictionary )launchOptions{
self.myWindow = [[UIWindow alloc] initWithFrame:[[UIScreen mainScreen] bounds]];
self.myWindow.backgroundColor=[UIColor whiteColor];
self.myWindow.rootViewControl ＝ rootViewControl;
[self.myWindow makeKeyAndVisible];
return YES;
}

UIWindowLevel

UIWindow有三个层级，分别是Normal，StatusBar，Alert。打印输出他们三个这三个层级的值我们发现从左到右依次是0，1000，2000，也就是说Normal级别是最低的，StatusBar处于中等水平，Alert级别最高。

通常我们的程序的界面都是处于Normal这个级别上的，系统顶部的状态栏应该是处于StatusBar级别，UIActionSheet和UIAlertView这些通常都是用来中断正常流程，提醒用户等操作，因此位于Alert级别。

KeyWindow

keyWindow是指定的用来接收键盘以及非触摸类的消息，而且程序中每一个时刻只能有一个window是keyWindow。

UIView

UIView表示屏幕上的一块矩形区域，它在App中占有绝对重要的地位，因为IOS中几乎所有可视化控件都是UIView的子类。负责渲染区域的内容，并且响应该区域内发生的触摸事件
UIVIew中的内容主要要依赖于CALayer，CALayer以后会记录
UIView的功能
1.管理矩形区域里的内容
2.处理矩形区域中的事件
3.子视图的管理
4.还能实现动画

UIView常用属性

frame： 相对父视图的位置和大小
bounds：相对自身的位置和大小，所以bounds的x和y永远为0
center：子视图的中点坐标相对父视图的位置
transform:可以通过这个属性控制视图的放大缩小和旋转
superview：获取父视图
subviews：获取所有子视图
alpha：视图的透明度（0－1）
tag：视图的标志，设置了tag后，可以通过viewWithTag方法拿到这个视图
userInteractionEnabled：是否响应用户事件

IOS多线程编程

By yanlc

发表于 2015-08-31

IOS有三种多线程使用方式

NSThread

NSThread是轻量级的封装层次比较低的，所以需要自己管理线程生命周期、线程同步
创建NSThread
+ (void)detachNewThreadSelector:(SEL)aSelector toTarget:(id)aTarget withObject:(id)anArgument
- (id)initWithTarget:(id)target selector:(SEL)selector object:(id)argument
参数 target selector消息发送对象 selector 线程要执行的方法 argument传输给target的参数

1、[NSThread detachNewThreadSelector:@selector(doSomething:) toTarget:self withObject:nil];
2、NSThread* myThread = [[NSThread alloc] initWithTarget:self selector:@selector(doSomething:) object:nil];
[myThread start];

第一种方式会直接创建线程并且开始运行线程，第二种方式是先创建线程对象，然后在start这个和java的线程操作就很像了

NSObject也提供了两种线程操作
1、- (void)performSelectorInBackground:(SEL)aSelector withObject:(id)arg
创建一个线程在子线程执行，aSelector代表了新创建的线程，arg是传入的参数
2、- (void)performSelectorOnMainThread:(SEL)aSelector withObject:(id)arg waitUntilDone:(BOOL)wait;
该方法的作用是在主线程中，执行制定的方法（代码块）
NSObject类的performSelectorOnMainThread和performSelectorInBackground可以实现简单的多线程编程技术

NSOperation

使用NSOperation有三种使用方式
1，使用它的子类NSInvocationOperation

NSInvocationOperation *operation=[[NSInvocationOperation alloc]initWithTarget:self selector:@selector(test) object:nil];
[operation start];
2，使用子类NSBlockOperation

NSBlockOperation *operation=[NSBlockOperation blockOperationWithBlock:^{
  //......
}];

[operation addExecutionBlock:^{
      //....
 }];
[operation start];

上面的调用start都是同步执行的，可以使用将operation添加到NSOperationQueue中进行异步执行
NSOperationQueue * queue=[[NSOperationQueue alloc]init];
//把操作添加到队列中
//第一种方式
[queue addOperation:operation1];
[queue addOperation:operation2];
[queue addOperation:operation3];
//第二种方式
[queue addOperationWithBlock:^{
NSLog(@”NSBlockOperation3—4——%@”,[NSThread currentThread]);
}];

3，自定义NSOperation，这种方法和java中的runnable非常相似
第一步新建一个类继承NSoperation
第二步实现main方法，main方法就是写自己的线程逻辑和java的run方法一样

GCD

GCD是苹果公司为多核的并行运算提出的解决方案,会自动利用更多的CPU内核（比如双核、四核）会自动管理线程的生命周期（创建线程、调度任务、销毁线程）

// 后台执行：
dispatch_async(dispatch_get_global_queue(0, 0), ^{
// something
});
// 主线程执行：
dispatch_async(dispatch_get_main_queue(), ^{
// something
});
// 一次性执行：
static dispatch_once_t onceToken;
dispatch_once(&onceToken, ^{
// code to be executed once
});
// 延迟2秒执行：
double delayInSeconds = 2.0;
dispatch_time_t popTime = dispatch_time(DISPATCH_TIME_NOW, delayInSeconds * NSEC_PER_SEC);
dispatch_after(popTime, dispatch_get_main_queue(), ^(void){
// code to be executed on the main queue after delay
});
// 自定义dispatch_queue_t
dispatch_queue_t urls_queue = dispatch_queue_create(“blog.devtang.com”, NULL);
dispatch_async(urls_queue, ^{
　 　// your code
});
dispatch_release(urls_queue);
// 合并汇总结果
dispatch_group_t group = dispatch_group_create();
dispatch_group_async(group, dispatch_get_global_queue(0,0), ^{
// 并行执行的线程一
});
dispatch_group_async(group, dispatch_get_global_queue(0,0), ^{
// 并行执行的线程二
});
dispatch_group_notify(group, dispatch_get_global_queue(0,0), ^{
// 汇总结果
});

GCD的另一个用处是可以让程序在后台较长久的运行。
在没有使用GCD时，当app被按home键退出后，app仅有最多5秒钟的时候做一些保存或清理资源的工作。但是在使用GCD后，app最多有10分钟的时间在后台长久运行。这个时间可以用来做清理本地缓存，发送统计数据等工作。让程序在后台长久运行的示例代码如下：

// AppDelegate.h文件
@property (assign, nonatomic) UIBackgroundTaskIdentifier backgroundUpdateTask;
// AppDelegate.m文件
- (void)applicationDidEnterBackground:(UIApplication *)application
{
[self beingBackgroundUpdateTask];
// 在这里加上你需要长久运行的代码
[self endBackgroundUpdateTask];
}
- (void)beingBackgroundUpdateTask
{
self.backgroundUpdateTask = [[UIApplication sharedApplication] beginBackgroundTaskWithExpirationHandler:^{
[self endBackgroundUpdateTask];
}];
}
- (void)endBackgroundUpdateTask
{
[[UIApplication sharedApplication] endBackgroundTask: self.backgroundUpdateTask];
self.backgroundUpdateTask = UIBackgroundTaskInvalid;
}

Studio导入github中开源项目

By yanlc

发表于 2015-07-27

github是全球最大的代码托管网站，里面有大量的高质量的开源项目源码，作为一名android开发的码农，学习github上面的优秀代码是很有必要的。现在github上面大部分的android项目已经都是用gradle构建的Android Studio项目，由于gradle和Android studio更新的速度比较快，很多项目的gradle版本都不同，这就为项目导入带来了很多困扰，出现大量问题,接下来我们来解决这些问题

第一步

下载一个github的android开源项目,这里我选择的是fresco的使用demo,Fresco是Facebook最新推出的一款用于Android应用中展示图片的强大图片库，可以从网络、本地存储和本地资源中加载图片,为了节省数据和CPU，它拥有三级缓存,而当图片从屏幕上消失时，会自动释放图片所占的内存.是一个非常棒的图片库

git clone https://github.com/liaohuqiu/fresco-demo-for-gradle.git 下载到本地，我们来看一下他的目录

源码有了，在导入项目之前我们需要修改一下配置文件

第二步

1，如果项目的gradle的版本与本地gradle环境的版本不同，在导入项目之后，Android Studio会下载项目gradle的版本，国内的下载速度可想而知，不翻墙的话，速度是非常慢的甚至下载失败，所以首先修改根目录里面的gradle版本下载链接

项目根目录gradle->wrapper->gradle-wrapper.properties

内容是

distributionBase=GRADLE_USER_HOME
distributionPath=wrapper/dists
zipStoreBase=GRADLE_USER_HOME
zipStorePath=wrapper/dists
distributionUrl=https\://services.gradle.org/distributions/gradle-2.2.1-all.zip

修改最后一行gradle-2.2.1-all.zip与本地gradle版本一致，不清楚的可以在Android Studio中新建一个project查看里面的gradle-wrapper.properties文件
2，修改根目录中的build.gradle

修改classpath “com.android.tools.build:gradle:${ANDROID_GRADLE_PLUGIN_VERSION}”与本地版本一致，这个项目的版本号ANDROID_GRADLE_PLUGIN_VERSION在根目录的gradle.properties下面，修改它与本地一致，不知道版本号的可以新建项目查看里面的配置

3，修改samples->comparison下面的build.gradle内容

修改sdk编译版本等版本与本地一致

minSdkVersion 9
targetSdkVersion 21
minSdkVersion 9

Centos连接无线网

By yanlc

发表于 2015-07-26

今天将一台老电脑装了Centos系统，安装完之后是命令行界面，由于安装的时候没有配置网络环境，ifconfig命令没有需要从新装，首先配置有限网络

cd /etc/sysconfig/network-scripts/ 用vi打开里面的第一个文件，编辑内容
HWADDR=00:0C:29:8D:24:73
TYPE=Ethernet
BOOTPROTO=static #启用静态IP地址
DEFROUTE=yes
PEERDNS=yes
PEERROUTES=yes
IPV4_FAILURE_FATAL=no
IPV6INIT=yes
IPV6_AUTOCONF=yes
IPV6_DEFROUTE=yes
IPV6_PEERDNS=yes
IPV6_PEERROUTES=yes
IPV6_FAILURE_FATAL=no
NAME=eno16777736
UUID=ae0965e7-22b9-45aa-8ec9-3f0a20a85d11
ONBOOT=yes #开启自动启用网络连接
IPADDR0=192.168.0.168 #设置IP地址
PREFIXO0=24 #设置子网掩码 不要设置 255.255.255.0
GATEWAY0=192.168.0.1 #设置网关
DNS1=8.8.8.8 #设置主DNS，不要设置自己所在区的dns，我配置过，不行
DNS2=8.8.4.4 #设置备DNS

配置完之后重启网络服务

service network restart

如果网络没有连上 reboot重启

安装ifconfig

yum install -y net-tools*

安装完之后 ifconfig查看网卡信息，发现只有一个有线网卡enp8s0和一个回路lo
查啊查，发现要NetworkManager服务没有开，泪奔

service NetworkManager start

启动服务，终于多出了一块无线网卡，以防万一，确保此网卡接口启动执行命令

sudo ip link set wlan0 up

扫描附近的无线网络

sudo iw dev wlan0 scan | less

连接没有加密的网络

sudo iw dev wlan0 connect [网络 SSID]

连接没有加密的网络wep加密网络

sudo iw dev wlan0 connect [网络 SSID] key 0:[WEP 密钥]

连接WPA或WPA2协议的话比较麻烦，需要修改/etc/wpasupplicant/wpa_supplicant.conf文件，增加如下行

network={
ssid=”网络 ssid”
psk=”密码”
priority=1
}

配置文件修改完成后

sudo wpa_supplicant -i wlan0 -c /etc/wpa_supplicant/wpa_supplicant.conf

这里发现了冲突，google了一下发现wpa_supplicant和NetworkManager不能同时启动，否则会冲突，只能关闭NetworkManager

service NetworkManager stop

然后在

sudo ip link set wlan0 up
开启无线网卡

获取 IP 地址

sudo dhcpcd wlan0

Android Studio解决gradle和sdk更新问题

By yanlc

发表于 2015-07-25

之前一直用Eclipse作为开发工具，但随着Google的大力推动和Studio的不断完善，Studio的地位已经越来越高，github上大部分的项目都已经采用了Studio作为开发工具，为了更好的学习，最近开始接触使用Studio。
Android Studio作为一个官方推出的开发工具确实有其优势，集成了源码库，一行库引用就可以加载编译，确实简单实用，但作为一名天朝的开发人员，我只能呵呵了，初用Studio的感觉就是这都是什么鬼 Maven连不上 JCenter连不上 Android Studio无法升级 Gradle无法同步，顿时感觉一万头羊驼向你蹦来
看来想用好Studio第一步就是要解决翻墙问题

Host

第一个想到的解决办法就是修改Host,记得原来收藏过一个不断更新Host的网址，翻出来http://levi.yii.so/archives/3553，该博客博主乱炖不仅定期更新host文件和一些免费的VPN，而且也包含了很多技术方面的知识，你值得收藏。
host修改的优点是简单方便，缺点需要不断的更新

SDK国内镜像站

说道这里就不得不提androiddevtools,绝对是天朝Android开发人员必收藏的网址，里面包含了Android开发所需的各种开发工具和sdk更新镜像地址。使用国内镜像地址可以解决sdk更新问题，配置方法在androiddevtools里面可以找到。
但是gradle的同步和JCenter库连不上不能解决

红杏翻墙

红杏翻墙提供了一个公益代理http://hx.gy:1080，该代理只包含开发人员所需的域名，所以只能作为开发人员专用，这是红杏免费提供给开发人员。红杏公益代理可以解决Android Studio开发更新的所有问题。

  1. github加速

    git config —global http.proxy http://hx.gy:1080

  2. 更新sdk

    Android SDK Manager Tools->options设计红杏代理

  3. 设置Android Studio代理，解决Jcenter和Gradle更新同步

    setting->Appearance&Behavior->system setting->Http proxy 设置红杏代理，这样就可以下载gradle或者其他依赖了。

ps.除了公益代理，红杏翻墙也是非常不错的，价格便宜、使用稳定方便,速度快，推荐使用。推荐链接http://honx.in/_Va72OSZDj1034LfB
